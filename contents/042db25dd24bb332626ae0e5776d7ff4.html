Aprovar Alterações

UMLUseCase
Projeto2 :: Use Case Diagram :: Aprovar Alterações

Description

Fluxo principal

  1. Um segundo operador é requisitado para aprovar as alterações realizadas, requisitando o gestor da planta para análise final.

  2. O gestor confirma as alterações que lhe foram solicitadas e registra no sistema.

Fluxo Alternativo 1 Operador nega as alterações

  1. Um segundo operador é requisitado para aprovar as alterações realizadas.

  2. O segundo operador registra as suas observações no sistema e rejeita as alterações.

  3. O gestor analisa as alterações e aprova ou rejeita as alterações

Fluxo Alternativo 2 Gestor nega as alterações

  1. Um segundo operador é requisitado para aprovar as alterações realizadas.

  2. O segundo operador registra as suas observações no sistema e aceita ou rejeita as alterações.

  3. O gestor analisa as alterações e rejeita as alterações, registrando no sistema
